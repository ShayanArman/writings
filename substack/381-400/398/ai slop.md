I know what ai slop is to me.

Reusing code that could be refactored into their own components.

Rewriting logic again that is already elsewhere in the application.

Things like that are slop. Just writing code because ‘thought’ has become cheap and easy.

But this isn’t beautiful. And its confusing, as the more code paths, the more complexity is introduced.

For example two methods that run the same logic, now we have to update the mistake in 2*(all the places it they are used)! Rather if we had 1, that is actually half the total amount of possible ‘error spaces’.

So ya, less code is actually a good heuristic for better system design!
